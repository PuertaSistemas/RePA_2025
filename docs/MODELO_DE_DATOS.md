# Modelo de datos RePA 2025

## Modelo de datos sugerido

---

### **1. Diagrama de Datos**  
```mermaid

erDiagram
    User {
        string id PK
        string email UK
        string hashed_password
        boolean is_active
        datetime created_at
        datetime last_login
    }

    Role {
        int id PK
        string rol UK
    }

    UserRole {
        int id PK
        string user_id FK
        int role_id FK
    }

    User ||--o{ UserRole : has
    Role ||--o{ UserRole : has
    UserRole }|--|| User : references
    UserRole }|--|| Role : references
```

## Diccionario de Datos

### Tabla: `users`

| Columna          | Tipo        | Clave Primaria | Nulable | Único | Descripción                                                                                   |
| :--------------- | :---------- | :------------- | :------ | :---- | :-------------------------------------------------------------------------------------------- |
| `id`             | `String`    | Sí             | No      | Sí    | Identificador único del usuario (UUID).                                                       |
| `email`          | `String`    | No             | No      | Sí    | Dirección de correo electrónico del usuario.                                                  |
| `hashed_password`| `String`    | No             | No      | No    | Hash de la contraseña del usuario.                                                          |
| `is_active`      | `Boolean`   | No             | No      | No    | Indica si el usuario está activo.                                                            |
| `created_at`     | `DateTime`  | No             | No      | No    | Fecha y hora de creación del registro del usuario.                                            |
| `last_login`     | `DateTime`  | Sí             | Sí      | No    | Fecha y hora del último inicio de sesión del usuario.                                         |

### Tabla: `roles`

| Columna | Tipo      | Clave Primaria | Nulable | Único | Descripción                      |
| :------ | :-------- | :------------- | :------ | :---- | :------------------------------- |
| `id`    | `Integer` | Sí             | No      | Sí    | Identificador único del rol.      |
| `rol`   | `String`  | No             | No      | Sí    | Nombre del rol (ej. administrador)|
